# Thai Receipt Generator

Generate varied Thai receipt/invoice PNG images for OCR testing. Produces combinatorial variations across line-item features (VAT, WHT, discounts) and visual templates.

## Setup

```bash
# Create venv and install
python3 -m venv .venv
source .venv/bin/activate
pip install -e .

# Install browser for rendering (one-time)
playwright install chromium
```

## Quick Start

```bash
# 1. Copy and edit config
cp .env.example .env

# 2. Generate (reads config from .env, no flags needed)
thai-receipt-gen gen

# Or override with flags
thai-receipt-gen gen --max-combos 50

# See how many total combinations exist
thai-receipt-gen list-combos
```

Images are saved to `./output/` by default.

## Configuration

All settings can be configured via `.env` file, environment variables, or CLI flags. Priority: **CLI flags > env vars > defaults**.

Copy `.env.example` to `.env` and uncomment the values you want to change:

```bash
cp .env.example .env
```

### `.env` example

```bash
RECEIPT_ITEMS=1,2
RECEIPT_TEMPLATES=formal_invoice,thermal_pos
RECEIPT_MAX_COMBOS=50
RECEIPT_SEED=42
RECEIPT_OUTPUT_DIR=output
RECEIPT_DPI=150
RECEIPT_NO_PER_ITEM_WHT=true
RECEIPT_NO_OVERALL_WHT=true
```

## Features & Variation Axes

Each receipt is generated by combining these independent axes:

| Axis | Options | Env var | CLI flag |
| --- | --- | --- | --- |
| Per-item VAT | on / off | `RECEIPT_NO_PER_ITEM_VAT` | `--no-per-item-vat` |
| Per-item WHT | on / off | `RECEIPT_NO_PER_ITEM_WHT` | `--no-per-item-wht` |
| Per-item discount | none / absolute / percentage | `RECEIPT_NO_PER_ITEM_DISCOUNT` | `--no-per-item-discount` |
| Standalone discount rows | 0 / 1 | `RECEIPT_NO_STANDALONE_DISCOUNT` | `--no-standalone-discount` |
| Overall discount | none / absolute / percentage | `RECEIPT_NO_OVERALL_DISCOUNT` | `--no-overall-discount` |
| Overall VAT | on / off | `RECEIPT_NO_OVERALL_VAT` | `--no-overall-vat` |
| Overall WHT | on / off | `RECEIPT_NO_OVERALL_WHT` | `--no-overall-wht` |

**Standalone discounts** are separate rows (e.g. "ส่วนลดพิเศษ") with an absolute amount, not tied to any line item.

**Constraint:** overall VAT/WHT and per-item VAT/WHT are mutually exclusive to avoid double-taxation. The combinator filters these automatically.

## Templates

Three visual styles:

| Template         | Description                            |
| ---------------- | -------------------------------------- |
| `formal_invoice` | A4 tax invoice with full table columns |
| `simple_receipt` | Compact receipt with minimal columns   |
| `thermal_pos`    | Narrow 320px POS/thermal printer style |

## CLI Reference

### `thai-receipt-gen gen`

Generate receipt images from combinatorial variations.

```bash
thai-receipt-gen gen [OPTIONS]

Options:
  --items TEXT           Comma-separated item counts (default: 1,2,3)
  --templates TEXT       Comma-separated template names (default: all three)
  --max-combos INT      Max combinations to sample (default: unlimited)
  --seed INT            Random seed for reproducible sampling (default: 42)
  --output-dir PATH     Output directory (default: ./output)
  --dpi INT             Rendering DPI (default: 150)
  --no-per-item-vat     Disable per-item VAT axis
  --no-per-item-wht     Disable per-item WHT axis
  --no-per-item-discount    Disable per-item discount axis
  --no-standalone-discount  Disable standalone discount axis
  --no-overall-discount     Disable overall discount axis
  --no-overall-vat          Disable overall VAT axis
  --no-overall-wht          Disable overall WHT axis
```

### `thai-receipt-gen list-combos`

Dry run — count combinations without rendering. Same options as `gen`.

## Examples

```bash
# Only single-item receipts, formal template, 20 samples
thai-receipt-gen gen --items 1 --templates formal_invoice --max-combos 20

# Two-item receipts with no WHT variations
thai-receipt-gen gen --items 2 --no-per-item-wht --no-overall-wht

# All templates, high DPI, custom output dir
thai-receipt-gen gen --max-combos 100 --dpi 300 --output-dir ./hires_output

# Check total count before generating
thai-receipt-gen list-combos --items 1,2
```

## Output Filenames

Each file is named by its variation axes:

```text
i2_vat1_wht0_dp-vat0_wht1_da_sd1_odn_ovat0_owht0_formal_invoice.png
│  │         │  │         │  │   │   │      │
│  └─item 1──┘  └─item 2──┘  │   │   │      └─ overall WHT off
│                             │   │   └─ overall VAT off
│                             │   └─ overall discount: none
│                             └─ 1 standalone discount
└─ 2 items
```

Per-item codes: `vat1`/`vat0`, `wht1`/`wht0`, `dn`(none)/`da`(absolute)/`dp`(percentage)
